# Commit Skill
1. Run `pwd` to confirm working directory
2. Run `git status` to show changes
3. Stage only the files I specify (or all if I say "all")
4. Write a concise commit message in conventional commit format
5. Do NOT create any summary docs or markdown files
6. Push only if I explicitly say "and push"

## CRITICAL: Commit Message Rules
- NEVER include "Co-Authored-By: Claude" or any Claude/Anthropic authorship attribution
- NEVER add any AI authorship lines to commit messages
- The commit message must contain ONLY the conventional commit subject (and optional body)
- No trailers, no co-authors, no signatures
